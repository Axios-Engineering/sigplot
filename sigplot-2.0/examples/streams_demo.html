Understand the Code

This code includes a file called "dsp.js" which can be found here. Read through this document to gain a better understanding of some of the functions used to control these plots.

Once some of the dsp variables are declared, the data layers for the plots are declared using overlay_array with null data and overlay_pipe.

The two plots are then set to mimic each other using the plot.mimic() method.

The update function generate ocsillations using dsp functions, and reloads data to the layers of the 1D plot using plot.reload([layer], [data]);. To update the data in the 2D plot, the plot.push([layer], [data]); method is used.

The window.onload function initializes the plot and enables the possibility to use dom elements called framerate, freq1, and freq2 to get input from the user and update the framerate, and frequency of the two oscillations on the 1D plot. This option is not implemented in this example.

In this same function, other dom elements have onclick functions that perform different actions on the plot. Click these elements (appear below the plot) to see how they change they change the plots.

On the 1D plot, an mtag listener is added, which will move the peak of one of the functions plotted when the mouse is clicked.

On the 2D plot, an mtag listener is added, which will add an annotation that reads "click" wherever the mouse is clicked on the plot.
